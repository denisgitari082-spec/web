Privacy Policy

Transparency on how we protect your financial data.

Effective Date: January 02, 2026

1. Our Privacy Core Principle

KwikQR is built on the principle of Zero Data Collection. The application is a local-only utility tool. We believe that your financial habits and M-Pesa transaction details are your business, and yours alone.

2. Sensitive Permissions Disclosure

To provide automation services, KwikQR requires the following sensitive Android permissions:

Accessibility API Purpose: We use this API to interact with system payment dialogs (USSD/STK Push). It allows the app to read the screen content to identify payment fields and simulate user clicks to complete the transaction process on your behalf.

Data Handling: This API does not record your screen, log your PIN, or transmit any data to external servers. It operates strictly in real-time on your device.

SMS Permission Purpose: KwikQR reads incoming M-Pesa confirmation messages to extract transaction details (Amount, Recipient, Transaction Code). This is required to automate record-keeping and verify payment status.

Data Handling: SMS data is processed locally in the app's secure sandbox. We do not access personal messages or non-financial SMS.

3. Data Storage & Security

  • Local Storage: Any transaction history you see in the app is stored in a local Hive Database on your device.
  • No Cloud Sync: We do not use Firebase, AWS, or any cloud providers. If you delete the app, your data is gone forever.
  • Encryption: We rely on the Android System Sandbox to prevent other apps from accessing KwikQR's local database.

4. Third-Party Sharing

KwikQR does not share, sell, or rent any user information to third parties, advertisers, or data brokers. There are no third-party SDKs (like Facebook or Google Analytics) bundled in this application.

As an Open Source project under the GPLv3 License, our logic is transparent. You can audit our source code at any time on GitHub.
